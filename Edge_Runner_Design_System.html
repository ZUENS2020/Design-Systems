EDGE RUNNER

Edge Runner

Concept: 镜之边缘风格,丰富简约动效 — Mirror's Edge style, rich yet minimal motion.

A dual-theme (light + dark) design system inspired by the visual language of Mirror's Edge: stark, near-monochrome cities punctuated by a single saturated accent — the "Runner Red." Geometry is sharp, space is generous, motion is purposeful.

Design Pillars

  • Clarity over decoration. White or near-white surfaces, high contrast, no gratuitous gradients.
  • One signal color. Runner Red (#E62020) is reserved for the single most important element on any screen. Everything else is neutral.
  • Flow, not bounce. Motion uses long easing curves and directional slides; never springy overshoot.
  • Architecture as composition. Hard 90° geometry, thin rules, and pristine negative space. Grids are visible, not hidden.

Core Specifications

Colors: Ink (#0B0B0D), Paper (#FAFAF7), Runner Red (#E62020), Sky (#3AA7FF)

Typography: Heebo (display/UI), JetBrains Mono (code/data)

Spacing: 4px base unit with 10-step scale

Motion: cubic-bezier(0.2, 0.7, 0, 1) with 4 duration tokens

Borders

  • 1px hairlines in neutral-200 for subtle divisions
  • 2px in Ink or Red for emphasis
  • No glows. No colored shadows except shadow-ping.

Backgrounds & Imagery

  • Flat color first. Paper or Ink fills 95% of surfaces.
  • No gradients as decoration. One acceptable gradient: horizontal Runner-Red → transparent on nav underline.
  • Rooftop imagery (full-bleed architecture) permitted for hero moments only — always cool-toned, high-key, 40% empty sky minimum.
  • No repeating textures. No illustrations. No emoji.
  • Photography: Cool temperature, high-key, architectural or sky-forward, ≥40% empty space for type.

Transparency & Blur

  • Blur: Used only for fixed top nav over scrolling: backdrop-filter: blur(12px) + 85% Paper fill.
  • Transparency: Limited to focus-ping (20% red) and disabled states (40%).

Motion & Animation

Motion Tokens

--ease-runner: cubic-bezier(0.2, 0.7, 0, 1) ← signature curve, long tail --ease-cut: cubic-bezier(0.8, 0, 0.2, 1) ← decisive state changes --ease-std: cubic-bezier(0.4, 0, 0.2, 1) ← standard easing --dur-flash: 120ms ← quick feedback --dur-step: 240ms ← UI transitions --dur-run: 480ms ← primary motion --dur-glide: 720ms ← slow reveal

Motion Rules

  • Clipped reveals: clip-path: inset(0 100% 0 0) animating to inset(0 0 0 0) — text swipes in from left.
  • Directional slides: translateX/Y of 16–24 px, never scale bounces.
  • Focus ping: 2px red underline scales from scaleX(0) to scaleX(1) with origin left on focus.
  • Never: Spring overshoot, rotating spinners with >1 revolution, wobble.
  • Hover states: opacity: 0.7 on icons/links, background darkens by 4% on buttons, 2px red bottom-border slides in on nav items. Duration --dur-step.
  • Press states: transform: translateY(1px) + background darkens 8%. Never scale.

Iconography

Icon System

Lucide Icons (https://lucide.dev) — loaded via CDN, rendered as inline SVG at runtime.

Specifications

  • Default size: 20 × 20 px
  • Stroke width: 1.5 (overrides Lucide's 2 default — matches the system's hairline language)
  • Color: currentColor — icons inherit from their text context
  • Fill: always none. Line-only. No filled icons.

Usage Example

<!-- CDN --> <script src="https://unpkg.com/lucide@latest/dist/umd/lucide.js"></script> <script>lucide.createIcons({ "stroke-width": 1.5 });</script> <!-- Inline use --> <i data-lucide="arrow-right" width="20" height="20"></i>

What NOT to Use

  • Emoji — never. Breaks the system's monochrome precision.
  • Decorative unicode (★ ♥ ✓) — never as UI icons.
  • Filled / duotone icons — never.
  • Icon fonts other than Lucide — never.

Acceptable Typographic Symbols

Used as typography, not iconography:

  • → ← ↑ ↓ — directional arrows in button labels, pagination
  • · — middle dot as CTA separator
  • — — em-dash for asides
  • ·01 ·02 — numeric section markers (set in mono)

Logo Usage

  • Mark: Single red right-chevron (>) — the universal "this way" signal.
  • Wordmark: "EDGE RUNNER" with chevron, geometric rendering.
  • Never: Tint, rotate, or distort. Minimum clear space = 1 chevron width.
  • Substitution note: No source codebase provided. Lucide is the closest match to the system's visual grammar (thin, geometric, open). Swap to a bespoke or licensed set when one is provided.

Voice & Tone

Content Fundamentals

Voice: Terse. Technical. Confident without swagger. Think a freerunner's playbook — short sentences, verb-first, no hedging.

Writing Guidelines

  • Casing: Title Case for display/eyebrows; sentence case for UI copy and body. ALL CAPS reserved for eyebrows, status pills, and key labels (tracking +0.08em).
  • Pronouns: Second person ("you") in product copy; first person plural ("we") only on brand/marketing pages.
  • Emoji: Never. This brand uses iconography and typography alone.
  • Numbers: Arabic numerals, even for small numbers ("3 runners online"). Use real units with thin spacing: 12 ms, 1920 × 1080.
  • Punctuation: En-dashes for ranges (08–12); em-dashes for asides — spaced — like this.

Empty States & Errors

Empty state: Nothing here yet. Drop a file, or press N to start a new run.
Error: Route blocked. The server didn't respond in time. Retry in 5 s.

What to Avoid

  • Corporate warmth ("We're so excited!")
  • Exclamation points outside success states
  • Vague qualifiers ("simply", "easily", "just")
  • Emoji or decorative symbols

Layout Conventions

  • Fixed elements: Top nav (64 px), optional left rail (72 px), optional right meta panel (320 px).
  • Eyebrows: Small ALL-CAPS labels sit 8 px above every headline on marketing surfaces.
  • Number markers: 01 · 02 · 03 precede major sections — a trademark of the system.
  • Corner HUD treatment: Important meta can sit flush in corners (top-right timestamp, bottom-left coordinates) in mono, small caps.
  • Alignment: Left-aligned copy is default. Center only for hero/marketing moments.
